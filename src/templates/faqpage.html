CampFire FAQ

WHAT CAN CAMPFIRE DO?

  • Helps turn your iRule log .txt files into flame graphs
  • Create general flame graphs and/or differential flame graphs
  • Provides type and value filter options
  • Provides archival option of flame graphs in either .svg and/or .png

WHAT IS THE DIFFERENCE BETWEEN A GENERAL FLAME GRAPH AND A DIFFERENTIAL FLAME GRAPH?

Differential flame graphs are colored to show the differences in execution time between 2 data sets. In general flame graphs, the colors provide no additional information. The colors exist only to improve readability of the flame graph.

WHAT INFORMATION CAN I GET FROM A FLAME GRAPH?

  • Flame graphs allow users to identify the execution time of function calls quickly and accurately
  • Flame graphs visualize code paths of uploaded iRule log files and allows users to navigate it easily
  • Differential flame graphs provide users the ability to compare differences between 2 sets of iRule profiler log data and shows the results of increased, decreased or no change in execution time of function calls
  • Flame graphs provides users the ability to search for a specific function call and will highlight the function call matches in magenta pink, along with displaying the percentage of how much that function call takes relative to the entire program in the right bottom hand corner

WHAT DO THE RECTANGLES REPRESENT?

WHY AM I GETTING THIS ERROR MESSAGE: 'Invalid input, no Flamegraph created'?

  1. Uploading an empty .txt file. Check your .txt file and make sure your iRule log files exist.
  2. Uploading a .txt file and checking filters that eliminate all the data. Your iRule log file might already have a layer of filtration where some type filters have been applied. When picking the filter options, there is a chance that the combination of chosen type filters results in an empty input.

WHY AM I GETTING THIS ERROR MESSAGE: 'UNABLE TO GENERATE FILE: BYTECODE must not be lowest level filtered. EVENT, RULE, and/or RULEVM must be filtered with bytecode. Input file also cannot contain only BYTECODE entries'?

You are getting this error message because our application does not allow users to select BYTECODE as the only type filter. In addition, it does not allow users to select BYTECODE without applying either EVENT, RULE, and/or RULEVM. The file input also cannot only contain BYTECODE entries.

WHY IS THE WINDOW REFRESHING BACK TO THE MAIN PAGE?

  1. Uploading a file that is not a .txt file. CampFire only accepts .txt files as input.
  2. Failing to upload a before and/or after .txt file when making a differential flame graph. To make a flame graph, a valid .txt file must be uploaded for both the before and after to create a differential flame graph.
  3. Failing to upload all necessary .txt file.
